# Beszel

Lightweight server monitoring. Hub (web UI + PocketBase) + per-node agent. Agents self-register against the hub via the WebSocket / Universal Token flow ‚Äî no SSH key handling, no per-host UI clicks.

## Architecture

| Component | Resource | Image | Notes |
|---|---|---|---|
| Hub | Deployment | `henrygd/beszel` | PocketBase + SQLite at `/beszel_data`. Exposed via Envoy Gateway at `beszel.<domain>`, gated by Cognito OIDC. |
| Agent | DaemonSet | `henrygd/beszel-agent` | One pod per node. `hostNetwork: true`, `hostPID: true`. Listens on `:45876`, dials hub on cluster DNS via `dnsPolicy: ClusterFirstWithHostNet`. |
| Auth | Universal Token | ‚Äî | Random token generated by the hub once and persisted in PocketBase. Mirrored into Akeyless; agents read it via `TOKEN` env. |

## Per-environment configuration

In `homelab-environments/<env>/values.yaml`:

```yaml
apps:
  beszel:
    enabled: true
    argocd:
      targetRevision: ~
      helm:
        values:
          generic:
            deployment:
              pvcMounts:
                data:
                  hostPath: /mnt/<storage>/apps/beszel/data
          agent:
            enabled: true
            # Container runtime socket. Optional ‚Äî host CPU/RAM/disk work
            # regardless; this only enables container-level stats.
            containerSocket: /var/snap/microk8s/common/run/containerd.sock  # microk8s
            # containerSocket: /var/run/docker.sock                          # Docker
            # containerSocket: ~                                             # host-only
```

Defaults that you usually don't override:

- `agent.hubUrl: http://beszel-service.argocd.svc.cluster.local:8090` ‚Äî in-cluster Service URL. Override only if the hub lives in a different cluster.
- `agent.tokenSecret: beszel-token` ‚Äî name of the K8s Secret produced by ESO.
- `agent.listen: "45876"` ‚Äî agent listen port.

## First-time setup

Per cluster. Roughly three minutes, then GitOps forever.

### 1. Create the host directory

```bash
ssh <user>@<node> 'sudo mkdir -p /mnt/<storage>/apps/beszel/data \
  && sudo chown 1000:1000 /mnt/<storage>/apps/beszel/data'
```

Path must match `pvcMounts.data.hostPath` in the env values. UID/GID `1000` matches the Beszel container user.

### 2. Let ArgoCD sync

The hub Deployment comes up. The agent DaemonSet will be in `CrashLoopBackoff` until the `TOKEN` secret exists ‚Äî this is expected for steps 3‚Äì4.

### 3. Generate the Universal Token in the hub UI

1. Open `https://beszel.<domain>`.
2. Create the admin account on first visit (just for hub access).
3. **Settings ‚Üí Tokens ‚Üí Enable Universal Token** ‚Üí copy the value.

### 4. Mirror the token into Akeyless

Path: `<global.akeyless.path>/beszel/UNIVERSAL_TOKEN`. Examples:

- `raspi.zimmermann.lat` ‚Üí `/raspi.zimmermann.lat/beszel/UNIVERSAL_TOKEN`
- `zimmermann.lat` ‚Üí `/zimmermann.lat/beszel/UNIVERSAL_TOKEN`

ExternalSecret refreshes hourly. To force-sync immediately:

```bash
kubectl -n argocd annotate externalsecret beszel-token-external-secret \
  force-sync=$(date +%s) --overwrite
```

Reloader restarts the agent pods when the underlying Secret changes. The agent dials `HUB_URL`, presents `TOKEN`, registers a System entry, and starts reporting metrics. No UI click per host.

## Adding another cluster

1. Enable the app in that env's values (same block as above).
2. Set `agent.enabled: true` and the correct `containerSocket`.
3. Create the host directory on the node(s).
4. Repeat steps 3‚Äì4 of "First-time setup" ‚Äî each hub has its own PocketBase, so its own Universal Token. The Akeyless path is namespaced by env (`<global.akeyless.path>/beszel/UNIVERSAL_TOKEN`), no collision.

## Operational notes

### What's on the PV

`/beszel_data/`:

- `data.db`, `data.db-shm`, `data.db-wal` ‚Äî PocketBase SQLite (users, system entries, tokens, fingerprints, ~30 days of telemetry).
- `auxiliary.db` ‚Äî secondary state.

Restic backs up the whole directory (`pvcMounts.data.backup.enabled: true`).

### Rotating the Universal Token

1. Hub UI ‚Üí **Settings ‚Üí Tokens ‚Üí Regenerate**.
2. Overwrite Akeyless `/beszel/UNIVERSAL_TOKEN` with the new value.
3. Force ExternalSecret refresh (command above); reloader restarts agents.

Already-registered systems keep working ‚Äî they have per-system fingerprints in PocketBase and don't rely on the universal token after first registration.

### Resetting everything

```bash
kubectl -n argocd scale deploy beszel-deployment --replicas=0

ssh <user>@<node> 'sudo rm -rf /mnt/<storage>/apps/beszel/data \
  && sudo mkdir -p /mnt/<storage>/apps/beszel/data \
  && sudo chown 1000:1000 /mnt/<storage>/apps/beszel/data'

kubectl -n argocd scale deploy beszel-deployment --replicas=1
```

Then redo "First-time setup" from step 3.

### Troubleshooting

| Symptom | Likely cause |
|---|---|
| Agent in `CrashLoopBackoff`, logs say `TOKEN` missing | Akeyless path not populated, or ExternalSecret hasn't synced yet. Force-sync. |
| Agent runs but hub shows "Keine Systeme gefunden" / "No systems found" | Token in Akeyless doesn't match the hub's current Universal Token. Re-copy after every regenerate. |
| Hub shows the agent but stats are pod-level, not host | `hostNetwork` / `hostPID` not applied ‚Äî `kubectl get ds beszel-agent -o yaml` and verify. |
| Agent can't resolve `HUB_URL` | `dnsPolicy: ClusterFirstWithHostNet` missing on the DaemonSet, or in-cluster DNS broken. |
| No container stats on microk8s | Expected ‚Äî Beszel reads Docker's API; containerd socket only provides host-level metrics. Host CPU/RAM/disk still work. |
| PV mount fails with "host path doesn't exist" | Run the `mkdir` from step 1. |

### Why WebSocket / Universal Token instead of SSH

Beszel supports two transports:

- **SSH (hub-initiated)** ‚Äî hub holds an SSH private key, SSHes into each agent on its known address. Requires per-system UI registration *and* a pre-shared SSH keypair. Two manual steps per agent.
- **WebSocket / Universal Token (agent-initiated)** ‚Äî agent dials hub with a token. Hub auto-creates the system entry on first connection. One manual step total (generate token once per hub), then every future agent self-registers.

This chart uses the WebSocket flow. See `apps/beszel/templates/agent-daemonset.yaml`.
